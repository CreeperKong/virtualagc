There is little known surviving Apollo-era documentation of the AURORA program, nor its closest relatives (SUNDIAL and SUNBURST), other than rare reference cards and meager memos.  Consequently, all of the information herein is derived from AURORA source code and analogies to less-closely-related software such as LUMINARY.

Go directly to word: 10 20 30 40

1a

I.D. word for this list.  Will contain 003218.

1b

Sync bits.  Will contain 004378.

2a,2b,3a

PIPAX, PIPAY, PIPAZ. These are CPU special registers (addresses 37-41 octal).  They contain the accumulation of output pulses from the X,Y, and Z accelerometers (a measure of the velocity changes), scaled (centimeters/second)/(5.85 × 214).

3b

Garbage. (Actually BMAGX, which is useless without BMAGY, BMAGZ. Emitted just because it follows PIPAZ in memory.)

4a

RADMODES. The descriptions of the bit positions were adapted from the "AURORA 85 & 86, SUNDIAL Mode Bits" document:

Bit
 Meaning
15
 Set 1 for RR continuous designate.
14
 Set 1 for RR antenna being remoded. I.e., remode requested or in progress. This bit is set whenever RRDESSM or RRDESNB determine that a designate may be performed after a remode has been done. It is cleared at the end of remoding by the REMODE routine.

The presence of this inhibits the GIMBAL Limit monitor. If this bit is set during a monitor reposition (see bit 11), the reposition will be interrupted by RR1AXIS and the remode will begin immediately.
13
 Set 1 for RR CDUs being zeroed.   Set by RR Automode monitor and by RR ZERO. Cleared by RR ZEROSB. Inhibits RR CDU Fail from lighting the tracker fail lamp.
12
 Set 1 for RR antenna in mode 2, set 0 for mode 1.  Set by RR ZEROSB according to trunnion angle: 0 for mode I (|T|<70); 1 for mode II (|T|>110°). Establishes limits for RR LIMCHK. Determines gimbal angles for remode and reposition sequences. Determines MODE A and MODE B setting in RR ANGLES (MODE A are gimbal angles if target sighted in present mode, etc.). May be inverted and reinverted (with rupt inhibited) by RR DESSM and RR DESNB to use RR LIMCHK to see if sighting possible after remode. Permanently inverted by REMODE.
11
 Set 1 for RR antenna out of limit.  Monitor Reposition in progress. This bit is set whenever the gimbal limit monitor finds T, S out of limits established for the present mode. It is set by the MONREPOS routine and cleared on completion of the reposition.

Its presence inhibits further operation of the gimbal limit monitor. If, during the reposition, the remode bit (14) is set (by a routine requesting a remode-designate operation), the reposition is interrupted (REPOSRPT) and the remode is started immediately. If, then, when a designate is feasible and STARTDES wants to initiate it, the presence of this bit says antenna control is in  progress, and the associated tasks will use bits 14, 11, and 10 to begin designation as soon as possible. 

If, during a designate operation this bit appears, the designate is terminated, since motion of the spacecraft may have driven the antenna out of limits or the antenna failed. The designate supervisor is given an error return (ROBADENO).
10
 Set 1 for RR designate or request in progress. This bit is set at STARTDES and cleared at the end of designation, successful or otherwise. If it is present at the end of a reposition, control is given to BEGDES to begin designation, possibly preceded by a remade operation. It is cleared on receipt of data good by the designate job to tell the designate task to go to RGOODEND.
9
 Set 1 for LR range on high scale, set 0 for low scale. Set each time A LT data is read from the LR. Used to detect scale changes so the sample which was attempted during the scale change may be discarded.
8
 Set 0 for LR velocity data failed. N samples could not be obtained from 2N tries. This bit in conjunction with Bit 5 lights tracker fail. Set and reset by data reading routines.
7
 Set 1 for RR CDU data OK, 0 for RR CDU fail — ignore if bit 2 and/or 13 = 1. Holds last sampled RR CDU fail bit. The absence of this and bits 2, 13 (auto & zeroing) lights tracker fail. Set/ reset by RRCDUCHK. 
6
 Set 0 for LR desired in pos 1, set 1 for pos 2. Desired LR Antenna position set by LR POSZ and fresh start. Used to monitor LR for proper position while reading data.
5
 Set 1 for LR alt. data fail. No LR Pas Data (See bit 8).
4
 Set 1 for RR data fail. No RR Data. N samples could not be taken in 2 N tries. Turns on tracker fail (see bits 8, 5).
3
 Set 1 for RR on high scale, 0 for RR range on low scale.  See bit 9.
2
 Set 1 for RR not in AUTO mode, 0 for RR in AUTO mode. Set/reset by RRAUTCHK in T4. If just off initiates RRTURNON (unless someone using RR). Its presence inhibits gimbal limit check. If just on, and some program using RR, (RRUSE) do a prog alarm.
1
 Set 1 for no monitor reposition alarm. Turn-on RRAUTCHK sequence. Set by RRTURNON and reset 1 sec after CDUs zeroed. Inhibits prog alarm in monitor reposition prog. Tested by RRZERO and if set, depends on RRTURNON to go to ENDRADAR after testing RR USE, which shows someone is waiting.

4b

SAMPLIM. A counter used to determine when LR radar positioning has taken too long.  Counts down once per second, starting at 14, and generating an error (005208) if becoming negative.

5a

OLDATAGD. A snapshot of bits 4, 5, and 8 of channel 33 — the RR, LR altitude, and LR velocity data-good discretes respectively — to indicate whether or not the old radar data is good.

5b

DESCOUNT. Single precision counter defining the maximum amount of time allowed for an attempt at radar designation, scaled B14 and unitless.

6

SAMPLSUM, +1. LR velocity data is smoothed by averaging 5 samples per 2-second period.  SAMPLSUM is the accumulator for the summation of the velocity data.

7

OPTYHOLD, +1.  A snapshot of the raw CDU angles (OPTY, OPTX).

8

TIMEHOLD, +1.  The time at which OPTY,OPTX were snapshotted into OPTYHOLD, +1.  (See word 7 above.)

9

ALT, +1. Initial/final altitude.

10a

ALTRATE. Single precision altitude rate calculated for display on the altitude rate meter, scaled B14 in units of Analog-altitude-rate-display bits.

10b

Garbage.  (Actually FINALT, but that's emitted later as well.  Here, it is emitted just because it follows ALTRATE in memory.)

11

ALTSAVE, +1. Double precision storage for previous altitude value for altitude meter display, scaled B28 in units of Analog-altitude-display bits.

12

FINALT, +1. Initial/final altitude.

13a,13b

FORVEL.  Single precision forward velocity component (Body coordinates) of the LM relative to the rotating moon, scaled B14 in forward velocity display units.

13b

LATVEL. Single precision lateral velocity component (Body coordinates; positive to the right when looking forward) of the LM relative to the rotating moon, scaled B14 in forward/lateral velocity units.

14a,14b,15a

CDUX (outer gimbal) CDUY (inner gimbal) CDUZ (middle gimbal): The actual values of the IMU CDU angles. Each register is an unsigned 15-bit fraction, scaled degrees/360, and is updated by counter interrupts. A middle gimbal angle greater than 70 degrees will light the Gimbal Lock lamp on the DSKY. Whenever the magnitude of the MGA exceeds 85 degrees the ISS stabilization loop will be disabled. As a result, the CDUZ register should maintain values that correspond to middle gimbal angles between -70 degrees and +70 degrees.

15b

Garbage. (Actually OPTY because it follows CDUZ in memory, but OPTY is explicitly emitted later.)

16a,16b

OPTY, OPTX. Current values of the CPU's special registers of those names.

17a,17b,18a,18b

13* MIDFLAG.  Set to 1 within orbital integration routine at beginning of time step when magnitude of conic position vector is greater than the constants RME (earth-centered) or RMM (moon-centered) and set to 0 if less than these constants.

19a

REDOCTR.  Counter for hardware restarts.  Set to zero by keyboard freshstart (Verb 36).  Incremented once per hardware restart by restart program.

19b

FAILREG. The most recent alarm code.

Refer to the Lunar Module Systems Handbook for AS-206/LM-1, Table 13-4 for a list of all possible commands, or Table 13-5 for more-detailed descriptions of some of them.

21a,21b

LASTYCMD, LASTXCMD.  RR TRUNNION and SHAFT ERROR COUNTERS.  Rendezvous Radar error counter commands; 21a is trunnion, 21b is shaft.  They are placed in the RR error counters and specify the rate at which the RR antenna is driven.  Values range from +384 to -384 and vary according to the angular error between the present and desired RR positions.  Calculated every 0.5 second in the Radar Designate Routine (DODES) whenever the the LGC is driving the RR antenna. A magnitude of 384 corresponds to a rate command of about 10 degrees/second.  The exact rate depends on the characteristics of the motors in the RR gyros.  21a and 21b are each scaled 2-14.

22

TANG, +1. Desired RR antenna trunnion and shaft gimbal angles, respectively.

23a,23b,24a

THETAD, +1, +2 (THETADX, THETADY, THETADZ). During normal RCS DAP operation, when the CMC Mode switch is in AUTO or HOLD and there are no RHC commands, these registers contain the desired current, (i,e., of this DAP cycle opposed to final) roll, pitch, and yaw CDU angles, treated as 15-bit unsigned fractions and scaled degrees/360, These quantities are used in the computation of phase plane attitude errors.

24b

Garbage. (Actually, DELVX because it follows THETAD +2 in memory.)

25a

MARKSTAT. The status of the marking process, where "marks" are navigation sightings of objects such as stars, taken with instruments such as the SXT (sextant).  The following description is adapted from the "Programmed Guidance Equations for LUMINARY 1B", and there may be errors I have not detected.

Single precision address storage and flagword. The address of the special working storage area assigned to the mark system is kept in bits 9-1 of MARKSTAT. Bit 15 is always 0. The other bits have the following significance:

Bit
 Meaning
15
 Unused.
14
  1. A pair of marks is in the process of being executed and redundant marks will not be accepted.
  2. A set of marks is complete and the next mark is to be interpreted as part of a new set.
13
  1. If a reject is entered, both marks will be cleared.
  2. A mark has been made since the last reject an it alone will be cleared if a reject is entered.
12
  1. The processing of marks is allowed.
  2. The processing of marks and mark rejects in the "MARKUP" routine is inhibited because the parameters needed for interpretation of marks have not yet been initialized, or the astronaut has indicated that he is finished taking marks.
11
  1. The Y mark of a pair has not yet been accepted.
  2. The Y mark of a pair has been accepted.
10
  1. The X mark of a pair has not yet been accepted.
  2. The X mark of a pair has been accepted.
9-1
 Unused.

25b

XYMARK. Single precision octal storage for bit indicating whether mark being processed is an X or a Y mark, or working storage for other occasions.  The values are TBD.

26-28

DRIFTO, +1, DRIFTI, +1, DRIFTT, +1.  According to PGE3:

  • DRIFTO is "value of gyro drift measurement output displayed in 'VALMIS', scale factor (assumed) BO, units 'radians', giving the 'vertical gyro drift'."
  • DRIFTI is "value of gyro drift measurement output displayed in 'TORQUE', scale factor (assumed) BO, units 'radians', giving the 'south gyro drift'."
  • DRIFTT is "Input 'drift' to gyro drift determination routine (to separate 'east gyro drift' from 'azimuth error'), scale factor BO, units radians. It has only its most significant half loaded by calling routines, with the least significant half set 0'."

29

TORQUE, +1. TBD.

30-32

ANGZ, ANGY, ANGX.  Values of determined angle changes about vertical, south, and east axes respectively, scale factor BO, units revolutions: they are 'azimuth alignment angle', 'south axis leveling angle', and 'east axis leveling angle' respectively (alpha, beta, gamma in official equation documentation).

33

VLAUN, +1. VLAUNy, VLAUNz: Values of 'horizontal velocity of launch vehicle' (due to sway) in north-south and east-west directions respectively, scale factor B9, units cm/sec.

34

Garbage.  (Specified in source code as VLAUN +4, +5, but those are unallocated addresses.)

35a,35b,36a,36b,37a,37b,38a,38b

RSTACK, +1, +6, +7, +12, +13, +18, +19.  RSTACKi: A series of 4 double-precision cells loaded with radar sample
data for display in nouns 66, 67 and (i = O, 2, 4, 6).  The Nominal List only downlinks a sampling of these values, whereas the High-Speed Radar List downlinks all of the values.

39a

UPLOCK. This is the uplink lock.  Only the least-significant bit is used.  When set to 1, further uplink activity is prevented.

39b

T6LOC. This does not appear to be used in SUNDIAL.  In AGC programs where it is used, it serves to prevent timer 6 interrupts (T6RUPT), which in turn prevents control of the RCS jets.
