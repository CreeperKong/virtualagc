7a,7b,8a

DSPTAB +11, +12, +13.   States of two — yes, two and not three — additional "rows" of DSKY relays, referred to as "C relays" because they relate collectively to 33 signals symbolized as C1, C2, ..., C33 that mostly are unrelated to the DSKY itself..

The C relays associated with word 8a (DSPTAB +13) are alternately referred to as "MCP relays".  The unmanned Apollo Command Module missions (AS-202, Apollo 4 & 6) had no astronauts aboard who could participate in controlling the spacecraft.  Part of that control was provided instead by the so-called Mission Control Programmer (MCP) device provided by North American Aviation.  For our purposes here, what's relevant is that the AGC could command activities in the spacecraft that it could not normally command in a manned mission, by writing to the MCP relays, which would then signal the actual MCP, which would then perform the desired operations in the spacecraft.

Each of DSPTAB +11, +12, +13 is partitioned into a 4-bit "relay word" (bits 15-12) and 11 bits of data (bits 11-1).  More detail about the C relays can be found in the "Apollo Command Module Block I Series 100 Guidance and Navigation System Manual", Figures 4-28, 4-231, 4-234, 4-208, and 4-240, among others, but here's a summary of their relay words and data bits:

  • 7a.  Relay Row 7, Relay Word 11002.
    1. C1, Zero encoder (nav-panel DSKY)
    2. C2, Coarse align (nav-panel DSKY)
    3. C3, Lock CDU (nav-panel DSKY)
    4. C4, Fine align (nav-panel DSKY)
    5. C5, Encoder zeroing (main-panel and nav-panel DSKYs)
    6. C6, CDU fail (main-panel and nav-panel DSKYs)
    7. C7, PIPA fail (main-panel and nav-panel DSKYs)
    8. C8, IMU fail (main-panel and nav-panel DSKYs)
    9. C9, Condition lamp test (nav-panel DSKY)
    10. C10, Attitude control lamp common (nav-panel DSKY)
    11. C11, Roll reentry (nav-panel DSKY)
  • 7b.  (Navigation panel only.)  Relay Row 13, Relay Word 11012 .
    1. C12, Zero optics CDUs
    2. C13, Turn star tracker on
    3. C14, Spare
    4. C15, Spare
    5. C16, Spare
    6. C17, Spare
    7. C18, Spare
    8. C19, Spare
    9. C20, Spare
    10. C21, Spare
    11. C22, Spare
  • 8a.  (Main panel only.)  Relay Row 13, Relay Word 11102.

8b,9a

TIME2, TIME1.  These are the values of the CPU's 14-bit timer registers TIME1 and TIME2 (memory addresses octal 36 and 37).  TIME1 increments every 10 ms.  It will wrap around from 037777 to 0 every 163.84 seconds.  TIME2 increments whenever wraps from 037777 to 0.  TIME2 and TIME1 together form a single 28-bit timer counting the number of centiseconds that have elapsed since power-up, and will itself wrap around to zero after just over 31 days.

9b

This is the value of the CPU register IN0 (address 0004), used principally to receive keyboard input from the DSKY, with the following bit assignments:

  • B1-B5 contains a keycode, if a keypad key is pressed, as follows.  Note that not all codes are used:
    • 00 (octal) for "no key pressed".
    • 20 (octal) for the digit 0.
    • 01-11 (octal) for the digits 0-9.
    • 21 for the VERB key.
    • 22 for the ERROR RESET key.
    • 31 for the KEY RLSE key.
    • 32 for the + key.
    • 33 for the - key.
    • 34 for the ENTER key.
    • 36 for the CLEAR key.
    • 37 for the NOUN key.
  • B6 is set when any of the B1-B5 or B15 is active, and is what triggers a "KEYRUPT" (keypad interrupt). 
  • B7-B13 are not used.
  • B14 is active when the AGC can be put into partially-powered-down "standby mode", and is, I believe, connected to a control-panel switch that's not on the DSKY.
  • B15 is active when the MARK key, not on the DSKY itself, is pressed, and activates B6 just like the DSKY-keypad keys do. It is used to cause the AGC to "mark" the current IMU data ... i.e., to snapshot it in memory.

It is perhaps worth noting that the KEYRUPT software expects for the entire lower 6 bits, B6-B1 to return to zero after the pushbutton is released, and will refuse to process additional keystrokes until this occurs.  At the same time, though, the way Virtual AGC DSKY simulation software works, it will not send a keycode to the AGC until the DSKY button is released.  This means that you can't use this downlink to detect keycodes, and even in the physical AGC/DSKY system you would have been very unlikely to be able to do so.

STCNTR.  Indicates which component of V76 update is presently being loaded.

25a,25b,26a

These are positions in which "MARK words", actually emitted asynchronously with respect to the preceding words in the first half of the downlist, are conveniently displayed as if emitted synchronously.  0-3 MARK words may have been emitted.  The possible values are:

  • 74000:  A placeholder used when less than 3 MARK words were emitted
  • 74001:  PIPA/PIPTIME MARK word
  • 74002:  Position/velocity MARK word
  • 74003:  CDUs/TFF MARK word

26b,27a,27b,28a,28b,29a,29b,30a,30b,31a,31b,32a,32b,33a

DSPTAB, +1, ..., +13.  DSKY and MCP relay data, as in words 1b-6b,7a-8a earlier, though with values that may have changed in the meantime.

33b,34a

TIME2GR, +1.  The AS-202 GSOP indicates that these are spares, but in Corona 261 source code it is the double-precision time at which Guidance Reference Release (GRR) occurred, i.e. the time at which the Saturn's guidance system began actively tracking the state vector.

34b,35a

TCUTOFF, +1.  Double precision time of GRR, lift-off, or engine on/off, whichever was the last to occur.

35b,36a,36b,37a,37b,38a,38b,39a,39b,40a,40b,41a,41b,42a

RAVEGON, +1, +2, +3, +4, +5, VAVEGON, +1, +2, +3, +4, +5, TAVEGON, +1.  Spacecraft state vector — obtained from the orbital-navigation program — to be used for the SPS2 burn.  RAVEGON et al. is a double-precision 3-vector containing the X, Y, and Z coordinates of the position, while VAVEGON contains the velocity, and TAVEGON contains the time.

42b,43a,43b,44a,44b,45a,45b,46a,46b,47a,47b,48a,48b,49a

STBUFF, +1, ..., +13.  1st through 14th components of an update via V76.

49b,50a,50b

MARK words emitted asynchronously during the 2nd half of the downlist.  See words 25a,25b,26a for a full explanation.
