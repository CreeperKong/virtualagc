There is no known surviving Apollo-era documentation of the SUNDIAL program, nor of its closest relative (AURORA), other than rare reference cards and meager memos.  Consequently, all of the information herein is derived from SUNDIAL source code and analogy to less-closely-related software such as COLOSSUS. 

Of particular help has been the document "Programmed Guidance Equations for COLOSSUS 3", which is an informal translation into pseudocode of the COLOSSUS 3 source code. Because this document will be referred to often, for conciseness I'll refer to it as PGE3.  Note, though, that units and/or scaling of quantities in SUNDIAL don't necessarily match those in PGE3.

Go directly to word: 10 20 30 40

1a

I.D. word for this list.  Will contain 017248.

1b

Sync bits.  Will contain 004378.

2a,2b,3a

PIPAX, PIPAY, PIPAZ. These are CPU special registers (addresses 37-41 octal).  They contain the accumulation of output pulses from the X,Y, and Z accelerometers (a measure of the velocity changes), scaled (centimeters/second)/(5.85 × 214).

3b,4a,4b

BMAGX, BMAGY, BMAGZ.  These are CPU special registers (addresses 42-44 octal).  The acronym BMAG stands for Body Mounted Attitude Gyros, which comprised a backup attitude reference.

5a,5b,6a,6b

PUSHLOC, PRIORITY, LOC, BANKSET. These are all parameters used by the executive.  The source code respectively refers to them as:

  • Word of packed interpretive parameters.
  • Priority of present job and work area.
  • Location associated with the job.
  • Usually contains BBANK setting.

7-9

DELVs. The sampled X, Y, & Z PIPA accumulations (velocity increments) with compensation for PIPA bias and scale factor errors. These quantities are in the stable member coordinate system and are scaled, (cm/sec)/(5.85 × 214.

10a,10b,11a,11b

SELFRET, SMODE, SFAIL, ERCOUNT.  These variables all relate to the built-in self-test (source code AGC_BLOCK_TWO_SELF_CHECK.agc).  Refer to document E-2065, "Block II AGC Self-Check and Show Banksum".  Here's a summary:

  • The options for the self-test are controlled by the number placed in SMODE (Noun 27), all numbers octal:
    • ±1:  Check all pulses possible by internal control of the computer.
    • ±2:  Check all IN/OUT instruction pulses.
    • ±3:  Check SC registers and all bit combinations.
    • ±4:  Check erasable memory.
    • ±5:  Check fixed memory.
    • ±6:  Extensive multiplication arithmetic check.
    • ±7:  Extensive division arithmetic check.
    • ±10:  Check everything in the previous 7 options.
    • ±11:  Turns on the electroluminescent displays in the DSKY.
    • -0:  Same as ±10, until an error is detected. 
    • +0:  Perform no check.  Instead, perform a back-up idle loop.
  • Testing continues indefinitely if no error is detected.  If an error is detected:
    • If SMODE is -0, testing resumes at the next test after the failed test.
    • For other negative SMODE, the failed test is restarted.
    • For positive SMODE, stop testing and begin a back-up idle loop.
  • SELFRET: A place where the self-test's return address is stored while displaying back checksums.
  • SFAIL: Address of the self-test routine that failed.
  • ERCOUNT: Number of failures detected.

13a,13b

DESOPTT, DESOPTS.  According to PGE3, these are "single precision values of desired CDUT and CDUS [trunnion and shaft CDUs] respectively, two's complement with same scaling as CDUT and CDUS respectively. They are used in 'OPTTEST' to generate optics driving commands (if OPTIND and SWSAMPLE are satisfactory), and in 'RATESUB' as the previous cycle's desired value for use in generating optics rate command information".

14a,14b,15a

CDUX (outer gimbal) CDUY (inner gimbal) CDUZ (middle gimbal): The actual values of the IMU CDU angles. Each register is an unsigned 15-bit fraction, scaled degrees/360, and is updated by counter interrupts. A middle gimbal angle greater than 70 degrees will light the Gimbal Lock lamp on the DSKY. Whenever the magnitude of the MGA exceeds 85 degrees the ISS stabilization loop will be disabled. As a result, the CDUZ register should maintain values that correspond to middle gimbal angles between -70 degrees and +70 degrees.

15b,16a

CDUT, CDUT. Note that this same item appears twice in succession. According to PGE3, CDUT is the "Single precision value of counter input cell 0035gj containing the optics CDU trunnion angle (also identified with the Y axis)".

16b

CDUS. According to PGE3, CDUS is the "single precision value of counter input cell 00368, containing the optics CDU shaft angle (also identified with the X axis)".

20b

SWSAMPLE. Apparently:

  • SWSAMPLE < 0 if OPT switch is on set to ZERO; otherwise:
  • SWSAMPLE = 0 if OPT switch is set to MAN(ual).
  • SWSAMPLE > 0 if OPT switch is set to CMC.

Aside:  Deducing the properties of SWSAMPLE directly from AGC source code, while not impossible, is rather too challenging for my tastes. Fortunately, SWSAMPLE appears also in COLOSSUS, and I found that easy way to understand it was to seek references in the pseudo-code found in the PGE3.  A still-easier method, had I kept searching, appears in document E-2456, "Apollo Guidance and Navigation Flow Charts, Program Colossus 2D", which says this about SWSAMPLE: "INDICATES WHETHER OPTICS SWITCH IS IN CMC, MANUAL, OR ZERO POSITION (BY POSITIVE, +0, OR NEGATIVE VALUE, RESPECTIVELY) AT PRESENT PASS THROUGH OPTMON".  The moral is never to throw away any documents.

21a

DESOPMOD.  According to PGE3, "single precision value of SWSAMPLE [see word 20b above] the previous time that "OPTMON" was performed, used to determine if a change has taken place in the setting of the optics mode switches (Computer Control, Manual, or Zero)".

21b

WTOPTION. According to PGE3, "single precision cell, scale factor B14, set to 11 in 'OPTMON' if switch from Zero to Manual optics mode with zeroing not yet complete (alarm 01168 is generated also). It is decremented by 1, with a lower limit of +0, when 'OPTMON' is entered with previous and present cycle modes both Manual.".

22a

IMUCADR. The following description is adapted from the "Programmed Guidance Equations for LUMINARY 1B", because I was unable to find any succinct, useful CM-related description, and there may be errors I have not detected.  Single precision storage for address to return to program that is making use of the ISS and waiting for a particular operation to be accomplished. However, I observe that special values such as +0, -0, and -1 may appear under conditions TBD as well.

22b

OPTCADR. The following description is adapted from the "Programmed Guidance Equations for LUMINARY 1B", because I was unable to find any succinct, useful CM-related description, and there may be errors I have not detected. Single precision octal storage for address to return to program that has requested marking and is waiting for the marking to be completed.

23a,23b,24a

THETAD, +1, +2 (THETADX, THETADY, THETADZ). During normal RCS DAP operation, when the CMC Mode switch is in AUTO or HOLD and there are no RHC commands, these registers contain the desired current, (i,e., of this DAP cycle opposed to final) roll, pitch, and yaw CDU angles, treated as 15-bit unsigned fractions and scaled degrees/360, These quantities are used in the computation of phase plane attitude errors.

24b

Garbage. (Actually, DESOPTT, which is already downlinked as word 13a.  Presumably, this downlink is repeated here simply because it follows THETAD +2 in memory.)

25a

MARKSTAT. The status of the marking process, where "marks" are navigation sightings of objects such as stars, taken with instruments such as the SXT (sextant).  The following description is adapted from the "Programmed Guidance Equations for LUMINARY 1B", because I was unable to find any succinct, useful CM-related description, and there may be errors I have not detected.

Single precision address storage and flagword. The address of the special working storage area assigned to the mark system is kept in bits 9-1 of MARKSTAT. Bit 15 is always 0. The other bits have the following significance:

  2. A mark has been made since the last reject an it alone will be cleared if a reject is entered.

25b

Garbage.  (Actually, the most-significant word of the double-precision quantity TIMESAV,+1, used to store the value of TIME2,TIME1 under certain circumstances.  However, only the most-significant word, useless by itself, is downlinked.  This is presumably simply because it accidentally follows MARKSTAT in memory.)

26-28

DRIFTO, +1, DRIFTI, +1, DRIFTT, +1.  According to PGE3:

  • DRIFTO is "value of gyro drift measurement output displayed in 'VALMIS', scale factor (assumed) BO, units 'radians', giving the 'vertical gyro drift'."
  • DRIFTI is "value of gyro drift measurement output displayed in 'TORQUE', scale factor (assumed) BO, units 'radians', giving the 'south gyro drift'."
  • DRIFTT is "Input 'drift' to gyro drift determination routine (to separate 'east gyro drift' from 'azimuth error'), scale factor BO, units radians. It has only its most significant half loaded by calling routines, with the least significant half set 0'."

GAZIMUTH.  TBD.

30-32

ANGZ, ANGY, ANGX.  According to PGE3, "values of determined angle changes about vertical, south, and east axes respectively, scale factor BO, units revolutions: they are 'azimuth alignment angle', 'south axis leveling angle', and 'east axis leveling angle' respectively (alpha, beta, gamma in official equation documentation)."

33

VLAUN, +1. According to PGE3,"VLAUNy, VLAUNz: Value of 'horizontal velocity of launch vehicle' (due to sway) in north-south and east-west directions respectively, scale factor B9, units cm/sec."

34-36

WPLATO, +1, WPLATI, +1, WPLATT, +1. According to PGE3,"Quantities computed in 'ALFLTd', scale factor BO, normalized by earth rotation rate (hence are converted to angles, for the one-second gyro drift evaluation cycle, by multiplication by Kgeorgk). The 'I' is south, the 'O' is vertical ('azimuth'), and the 'T' is east."

37-38

PIPAN, +1, PIPAE, +1.  TBD.  

39a

UPLOCK. This is the uplink lock.  Only the least-significant bit is used.  When set to 1, further uplink activity is prevented.

39b

T6LOC. This does not appear to be used in SUNDIAL.  In AGC programs where it is used, it serves to prevent timer 6 interrupts (T6RUPT), which in turn prevents control of the RCS jets.
